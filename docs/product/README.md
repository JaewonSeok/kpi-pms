# Product Docs Index

현재 성과관리/평가 제품 설계 문서는 아래 3개 축으로 정리되어 있다.

## Documents

- [KPI 성과관리 PRD](./performance-management-prd.md)
  - 페이지 목적, 사용자, 핵심 데이터, 우선순위 정의

- [페이지별 Low-Fidelity Wireframes](./page-wireframes-lowfi.md)
  - 각 페이지의 데스크톱/모바일 정보구조와 핵심 인터랙션

- [구현 준비 Backlog](./implementation-backlog.md)
  - API, DB, 엔티티, 컴포넌트, 전달 순서 기준

## Recommended Reading Order

1. PRD로 제품 목표와 역할별 요구를 이해한다.
2. Wireframe 문서로 화면 구조와 상태를 본다.
3. Backlog 문서로 구현 단위와 의존성을 확인한다.

## Recommended Execution Order

1. `평가 주기`
2. `조직 KPI`
3. `개인 KPI`
4. `월간 실적`
5. `평가 결과`
6. `이의 신청`
7. `등급 조정`
8. `조직도 관리`
