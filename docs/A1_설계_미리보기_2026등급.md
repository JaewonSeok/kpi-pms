## 1. 목적·범위·안전 원칙

### 1.1 목적
시연 동선 §3.8 (조직30%+개인70% 점수)·§3.9 (등급 자동산정)·§3.11 (결과 라이브) 빈틈을 한 페이지로 메운다. 5종 페르소나 가상 직원의 점수를 슬라이더로 조정하면 30:70 산식과 임계값 매칭으로 등급이 실시간 산정되어 표시된다.

### 1.2 범위 — 무엇을 한다 / 안 한다

| 영역 | 한다 | 안 한다 |
|---|---|---|
| 화면 | `/admin/preview-2026-grade` 신규 1페이지, 카드 5장. | 다른 화면 수정 X. |
| 데이터 | 가상 직원 5명을 **코드 상수**로 박음. | DB 조회 0 (employees/departments/department_score_intakes/evaluations 전부 비참조). |
| 계산 | 30:70 산식 + `calculateAbsoluteGrade2026` 호출. | 라이브 공식 `Evaluation.totalScore`/`gradeId` 쓰기 0. |
| 정책 | `EVALUATION_POLICY_2026.gradeThresholdGroups` 임계값 그대로 사용. | 정책 상수 수정 X. |
| 권한 | `ROLE_ADMIN`만. | 다른 역할 접근 차단. |

### 1.3 안전 원칙
- ★ **DB write 0**. `prisma.*.create/update/delete/upsert` 일절 없음. 로더는 코드 상수만 반환.
- ★ **공식 쓰기 가드 우회 시도 0**. `calculateAbsoluteGrade2026` / `calculateFinalPerformanceScore2026`은 pure function이므로 호출만으로 공식 쓰기 발생 안 함.
- ★ **스키마 변경 0**. `prisma/schema.prisma` 미터치.
- ★ **flag flip 0**. `evaluation2026Preview` default `true`만 활용 (이미 켜져 있음).
- ★ **라벨링**: 화면 상단에 "미리보기 전용 · 저장되지 않음" 배지 고정. 시연자/관객의 혼란 방지.

## 3. 신규 resolver — `calculateOrganizationPerformanceFromIntake2026`

### 3.1 위치
- **파일**: `src/lib/preview-2026-organization-score.ts` (신규)
- **export**: pure function. DB 조회 0. 부작용 0.

### 3.2 왜 새 함수가 필요한가 (`calculateFinalPerformanceScore2026` 미재사용 이유)

`calculateFinalPerformanceScore2026({ org, personal })`는 `org × 0.30 + personal × 0.70`을 적용한다. 즉 `org` 입력값은 **이미 가중 적용된 조직 30% 점수**여야 한다.

결정 (1)의 식:
- 팀원/팀장: `final = team × 0.20 + parent × 0.10 + personal × 0.70`
- DIV_HEAD: `final = division × 0.30 + personal × 0.70`

여기서 "조직 30% 점수" = `team × 0.20 + parent × 0.10` (팀원/팀장) 또는 `division × 0.30` (DIV_HEAD). 이 값을 `calculateFinalPerformanceScore2026`에 `org`로 넘기면 → `(team × 0.20 + parent × 0.10) × 0.30` 이 되어 **수치가 어긋남** (이중 가중).

따라서 A1 resolver는 **자체 식**으로 직접 합산한다. `finalScoreFormula`의 30/70 비율 상수만 참조 (decoupling).

### 3.3 입력·출력 타입

```ts
import type { EvaluationGrade2026RoleGroup } from '@/server/evaluation-grade-2026'
import type { OrganizationWeights2026 } from '@/lib/validations'

export type PreviewIntakeInput2026 = {
  // 결정 (1)의 분기 키. resolver는 'DIVISION_HEAD' 특례 vs 그 외로 갈림.
  roleGroup: EvaluationGrade2026RoleGroup  // 'TEAM_MEMBER' | 'TEAM_SECTION_LEADER' | 'DIVISION_HEAD'

  // 팀원/팀장만 사용. DIVISION_HEAD에선 무시.
  teamScore?: number

  // 팀원/팀장: 직속 상위 부서(실 또는 본부)의 점수.
  // DIVISION_HEAD: 본인 본부의 점수 (자기 division).
  parentScore: number

  // 본인 개인 평가 점수.
  personalScore: number

  // 직속 상위 부서 level. 팀원/팀장에서 withSection / withoutSection 가중치 분기.
  // DEFAULT 가중치는 양쪽 동일하므로 분기 영향 0. 단 커스텀 가중치 저장 시 분기 필요.
  // DIVISION_HEAD에선 무시.
  parentLevel?: 'SECTION' | 'DIVISION' | null

  // 활성 사이클의 OrganizationWeights2026. 미전달 시 DEFAULT_ORGANIZATION_WEIGHTS_2026 사용.
  weights?: OrganizationWeights2026
}

export type PreviewOrganizationScore2026Result = {
  // 가중 적용된 조직 30% 점수 (팀원/팀장: team*wTeam + parent*wParent / DIV_HEAD: division*0.30).
  organizationScore: number

  // 가중 적용된 개인 70% 점수 (personal * 0.70).
  personalScore: number

  // organizationScore + personalScore.
  finalScore: number

  breakdown: {
    roleGroup: EvaluationGrade2026RoleGroup
    teamScoreWeighted?: number      // 팀원/팀장만
    parentScoreWeighted?: number    // 팀원/팀장만
    divisionScoreWeighted?: number  // DIV_HEAD만
    personalScoreWeighted: number
    // 적용된 가중치 표시용 (UI 산식 풀이에 활용)
    appliedWeights: {
      team?: number
      parent?: number
      division?: number
      personal: number
    }
  }

  formulaVersion: typeof EVALUATION_POLICY_2026_VERSION  // '2026-PPT-PHASE0'
}
```

### 3.4 의사코드

```ts
import {
  EVALUATION_POLICY_2026,
  EVALUATION_POLICY_2026_VERSION,
} from './evaluation-policy-2026'
import { DEFAULT_ORGANIZATION_WEIGHTS_2026 } from './policy-2026-organization-weights'

// ★ DIVISION_HEAD 데모 특례: 본부×0.30 + 개인×0.70.
// 정책 schema(policy2026OrganizationWeights)에는 DIVISION_HEAD 전용 키 없음 — A1 코드 상수.
// 정책 담당자 정식 확정 후 schema 추가 검토 [확인 필요].
const DIVISION_HEAD_DIVISION_WEIGHT = 0.30
const DIVISION_HEAD_PERSONAL_WEIGHT = 0.70

function round1(n: number): number {
  return Math.round(n * 10) / 10
}

export function calculateOrganizationPerformanceFromIntake2026(
  input: PreviewIntakeInput2026
): PreviewOrganizationScore2026Result {
  // ── 분기 1: 본부장(DIVISION_HEAD) 데모 특례 ──
  if (input.roleGroup === 'DIVISION_HEAD') {
    const divisionWeighted = input.parentScore * DIVISION_HEAD_DIVISION_WEIGHT
    const personalWeighted = input.personalScore * DIVISION_HEAD_PERSONAL_WEIGHT
    return {
      organizationScore: round1(divisionWeighted),
      personalScore: round1(personalWeighted),
      finalScore: round1(divisionWeighted + personalWeighted),
      breakdown: {
        roleGroup: 'DIVISION_HEAD',
        divisionScoreWeighted: round1(divisionWeighted),
        personalScoreWeighted: round1(personalWeighted),
        appliedWeights: {
          division: DIVISION_HEAD_DIVISION_WEIGHT,
          personal: DIVISION_HEAD_PERSONAL_WEIGHT,
        },
      },
      formulaVersion: EVALUATION_POLICY_2026_VERSION,
    }
  }

  // ── 분기 2: 팀원/팀장 — 결정 (1)의 식 ──
  const weights = input.weights ?? DEFAULT_ORGANIZATION_WEIGHTS_2026

  // 직속상위 level로 withSection vs withoutSection 가중치 분기.
  // - parentLevel = 'SECTION' → withSection 가중치 (team=0.20, section=0.10, division=0.00).
  //   parentScore는 '실' 점수.
  // - parentLevel = 'DIVISION' → withoutSection 가중치 (team=0.20, division=0.10).
  //   parentScore는 '본부' 점수 (실이 없는 본부 직속).
  // DEFAULT는 양쪽 동일이라 분기 영향 0. 커스텀 저장 시 영향 발생.
  const teamWeight = weights.withSection.team  // 양쪽 동일 = 0.20 (DEFAULT)
  const parentWeight =
    input.parentLevel === 'DIVISION'
      ? weights.withoutSection.division  // 0.10 (DEFAULT)
      : weights.withSection.section       // 0.10 (DEFAULT) — parentLevel이 SECTION이거나 null이면 이쪽

  const teamWeighted = (input.teamScore ?? 0) * teamWeight
  const parentWeighted = input.parentScore * parentWeight
  const personalWeighted = input.personalScore * weights.personal

  return {
    organizationScore: round1(teamWeighted + parentWeighted),
    personalScore: round1(personalWeighted),
    finalScore: round1(teamWeighted + parentWeighted + personalWeighted),
    breakdown: {
      roleGroup: input.roleGroup,
      teamScoreWeighted: round1(teamWeighted),
      parentScoreWeighted: round1(parentWeighted),
      personalScoreWeighted: round1(personalWeighted),
      appliedWeights: {
        team: teamWeight,
        parent: parentWeight,
        personal: weights.personal,
      },
    },
    formulaVersion: EVALUATION_POLICY_2026_VERSION,
  }
}
```

- `parentScore` / `teamScore` / `personalScore`는 호출자(클라이언트)가 슬라이더 입력값으로 직접 제공.

### 4.3 계산 검산 (raw)

```

#2 팀장영업:    108×0.20 + 106×0.10 + 104×0.70
                = 21.6 + 10.6 + 72.8                 = 105.0 ∈ [100,110) → OUTSTANDING ✅

                = 22.4 + 11.0 + 77.0                 = 110.4 ∈ [105,115) → EXCELLENT ✅

                = 16.0 + 8.2 + 56.0                  = 80.2 ∈ [75,85)    → GOOD ✅
#5 본부직속팀원영업: 88×0.20 + 85×0.10 + 85×0.70
                = 17.6 + 8.5 + 59.5                  = 85.6 ∈ [80,90)    → NEED_IMPROVEMENT ✅
```

### 4.4 슬라이더 종류·범위 (직전 결정 O1 유지)

| 슬라이더 | 범위 | 적용 페르소나 |
|---|---|---|
| 개인 점수 — 비영업 | 0–100 | #3, #4 |
| 개인 점수 — 영업·본부장 | 0–130 | #1, #2, #5 |
| 조직 점수 (팀·실·본부) | 0–130 | 전 페르소나 (`department_score_intakes` CHECK와 일치) |

★ 비영업 개인 0~100 cap은 UI 제약. resolver/`calculateAbsoluteGrade2026`은 그 이상도 처리 가능하지만 시연 일관성을 위해 cap 적용.

### 4.5 TEAM_MEMBER_SALES 중첩 처리 (#5 영향 없음 확인)
- #5 점수 85.6은 SUPER 임계(≥110) 미만, OUTSTANDING `requiresPolicyConfirmation` 중첩 구간(≥110) 밖.
- → 단일 매칭(NEED_IMPROVEMENT [80,90))만 발생. `teamMemberSalesThresholdDecision` 미전달 OK.
- 만약 시연 중 슬라이더로 점수를 110 이상으로 올리면 `calculateAbsoluteGrade2026`이 `AMBIGUOUS_THRESHOLD_MATCH` 오류 반환할 수 있음. UI에서 그 시점에 "정책 담당자 확정 대기" 경고 배너 표시 — 보정.
